## Konsep dasar

Tiap **task** punya:

| Field | Default |
|---|---|
| `title` | required |
| `description` | optional, multi-line |
| `status` | `open` (initial) ‚Üí `in_progress` ‚Üí `review` ‚Üí `done` (atau `cancelled`) |
| `priority` | `medium` (default), pilihan: urgent / high / medium / low |
| `agentId` | optional ‚Äî task bound ke agent spesifik untuk dispatch |
| `tags` | array string, untuk filter/categorize |
| `cost` | accumulated USD cost dari turn-turn dispatch |
| `sessionId` | gateway session yang handle task ini (kalau sudah di-dispatch) |
| `projectId` | parent project (required) |

Transition di-trigger via:

- **Manual drag** di kanban ‚Äî UI POST update status
- **Skill `update_task.sh`** dari agent ‚Äî sub-agent / Master self-report
- **Auto** saat dispatch ‚Äî status auto `in_progress` saat task di-send ke gateway

## Dispatch ke agent

Klik tombol **Dispatch** di task panel. Yang terjadi:

- Task title + description dikirim ke agent assigned sebagai first message
- Agent kerja, status task otomatis pindah jadi `in_progress`
- Reply agent stream real-time ke task panel ‚Äî kamu bisa lihat progress

Kalau agent stuck atau butuh clarification, kamu bisa kirim follow-up message di task panel ‚Äî agent dapet konteks lengkap dari turn-turn sebelumnya.

## Multi-turn conversation per task

Tidak semua task selesai 1 kali jalan. Kamu bisa **re-dispatch** task yang udah `in_progress` untuk kasih:

- Clarification ("oh, maksud saya total per minggu, bukan per hari")
- Tambahan requirement ("plus bandingkan dengan Q2")
- Feedback hasil intermediate ("draft 1 OK, lanjut ke section 2")

Semua turn tersimpan di task panel ‚Äî kamu bisa scroll up untuk lihat history kerja agent dari awal sampai sekarang. Konteks **tidak hilang** antar turn.

## Apa yang agent lakukan saat handle task

Agent yang bekerja akan otomatis update progress di task panel kamu:

<Steps>
  <Step title="Mulai: tandai 'in_progress'">
    Saat agent mulai kerja, status task auto-update jadi **in_progress**. Kamu lihat aktivitas real-time di task panel.
  </Step>
  <Step title="Update progress di tengah jalan">
    Kalau task panjang, agent kasih comment progress berkala ‚Äî tanpa nunggu selesai. Mis. "Found 3 edge cases, lanjut investigate".
  </Step>
  <Step title="Blocked kalau stuck">
    Kalau agent butuh sesuatu (mis. credential, decision dari kamu), status pindah ke **blocked** + comment yang menjelaskan apa yang dia tunggu. Kamu dapat notifikasi.
  </Step>
  <Step title="Submit untuk review">
    Kalau agent merasa task selesai, dia upload **deliverable** (file output, link, dll) ke task ‚Üí status pindah **in_review**. Notifikasi muncul ke kamu.
  </Step>
  <Step title="Approve atau request changes">
    Kamu review hasil. Klik **Approve** kalau OK ‚Äî status jadi **done**. Atau klik **Request Changes** + tulis feedback ‚Äî status balik **in_progress**, agent re-dispatch dengan feedback kamu.
  </Step>
</Steps>

## ADLC Stage Kanban

Toggle dari "Status Kanban" ke "Stage Kanban" di Board view. Kolom-nya 8 ADLC phase:

```
Discovery ‚Üí Design ‚Üí Architecture ‚Üí Implementation ‚Üí QA ‚Üí Docs ‚Üí Release ‚Üí Ops
```

Tiap task bisa di-tag stage manual atau auto-detect dari role agent yang handle (mis. SWE in Implementation, QA in QA phase).

Useful untuk product team yang track lifecycle feature secara phase-based, bukan cuma todo/done.

## Filter & search

Toolbar Board:

- **Search bar** ‚Äî match title + description
- **Filter assignee** ‚Äî pilih agent atau "Unassigned"
- **Filter status** ‚Äî multi-select
- **Filter priority** ‚Äî urgent/high/etc
- **Filter tags** ‚Äî chip-based, AND logic

State filter persist di URL query ‚Üí bookmarkable.

## Cost tracking

Tiap turn dispatch, gateway report cost (token usage √ó model price). Auto-accumulate ke `task.cost`. Total project cost = sum task cost.

Useful untuk:
- Budget tracking per project
- Identify task yang expensive (mungkin perlu split atau pakai model lebih murah)
- Audit "berapa AOC bantu kerja kita minggu ini"

## Bulk action (planned)

Belum tersedia v1. Sementara harus per-task: assign, status change, delete.

## Mission Room ‚Üî Task

Saat task di-create di project yang punya default mission room:
- Notification posted di mission room (`New task: <title>`)
- Member room (Master + sub-agent) bisa lihat tanpa polling kanban
- Reply di room tidak otomatis link ke task ‚Äî gunakan TaskPanel untuk percakapan task-specific

Untuk room custom yang tidak terikat project: task tidak otomatis muncul di sana.
