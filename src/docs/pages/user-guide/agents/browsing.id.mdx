Setiap card menampilkan (top → bottom):

- **Hash ID** kiri-atas — `#XXXX` (4 char pertama dari agent ID, mono uppercase)
- **Status pill** kanan-atas — colored pill dengan dot + label (Active / Idle / Working / Paused / Error)
- **Avatar 64×64** di tengah — emoji + custom preset, hover bikin scale-105
- **Nama display** — bold, center-aligned
- **Role badge** — `[number]` color circle + emoji + uppercase label, color match role template (mis. `0 🧭 MASTER ORCHESTRATOR` ungu, `4 💻 SOFTWARE ENGINEER` cyan)
- **Vibe / description** — italic muted text, line-clamp-2 (kalau kosong tetap reserve 2 line height untuk grid alignment)
- **3-stat grid** — Sessions / Cost / Tokens, masing-masing dalam pill `bg-foreground/3` rounded
- **Channels row** (kalau ada) — icons Telegram/WhatsApp/Discord di bawah divider
- **Model footer** — mono small text, mis. `claude-sonnet-4-6`, dengan dot kecil di kiri

| Status | Kapan muncul |
|--------|---------------|
| **Active** | Agent online + gateway running + ready terima turn |
| **Idle** | Gateway running, tapi tidak ada session aktif |
| **Working** | Lagi process turn (animated pulse, hero shadow ber-glow amber) |
| **Paused** | User explicit pause via Edit Configuration |
| **Error** | Gateway crash atau stuck — restart dari Agent Detail |

Card juga punya **role accent strip 0.5px** di paling atas (gradient horizontal dari role color) sebagai visual anchor — match warna role template.

## Search & Filter

<Steps>
  <Step title="Search bar">
    Cari berdasarkan nama, ID, atau persona snippet. Real-time match — cukup ketik beberapa karakter.
  </Step>
  <Step title="Filter Owner (admin only)">
    Default: agent kamu sendiri. Dropdown:
    - **Me** — punya kamu (default)
    - **All** — semua user (cross-tenant view, admin only)
    - **User specific** — pilih user tertentu untuk impersonate view
  </Step>
  <Step title="Status filter (planned)">
    Filter by status (active/idle/processing/error) — belum tersedia di v1.
  </Step>
</Steps>

<Callout type="info" title="Default scope = 'me'">
Semua user (admin atau bukan) default-nya cuma lihat agent mereka sendiri. Ini disengaja untuk hindari accidental cross-tenant leakage. Admin harus eksplisit pilih `?owner=all` untuk lihat semua. Detail di <ConceptLink to="/docs/reference/sharp-edges">Sharp Edges</ConceptLink>.
</Callout>

## Klik card → buka detail

Card di `/agents` cuma punya **satu aksi**: klik di mana saja di area card untuk navigate ke `/agents/<id>` (detail page). Tidak ada context menu, dropdown, atau action overlay di card itu sendiri.

<Callout type="info" title="Semua aksi ada di Agent Detail page">
Edit konfigurasi, restart gateway, kelola channel, install skill, dispatch task, delete agent — semuanya hidup di **Agent Detail page**. Card di list cuma entry point. Lihat <ConceptLink to="/docs/user-guide/agents/detail-overview">Halaman Detail Agen</ConceptLink> untuk tour lengkap.
</Callout>

Hover effect:
- Card **lift up** 2px (`hover:-translate-y-0.5`)
- Shadow membesar (`hover:shadow-xl hover:shadow-black/20`)
- Avatar di hero scale-105 (subtle pop)
- Cursor jadi pointer — sinyal seluruh area click-able

## Provision agent baru

Klik tombol **+ New Agent** di kanan atas. Akan muncul `TemplateEntryModal` dengan dua pilihan:

<Tabs>
  <Tab label="ADLC Template">
    Pilih dari 9 role template (PM Discovery, PA Monitor, UX Designer, EM Architect, SWE, QA Engineer, Doc Writer, Biz Analyst, Data Analyst). Template auto-fill: emoji, color, persona description, recommended skills, soul prompt.
  </Tab>
  <Tab label="Blank">
    Mulai dari kosong. Kamu isi semua field manual. Cocok kalau peran agent gak match template existing.
  </Tab>
</Tabs>

Detail flow: <ConceptLink to="/docs/user-guide/agents/provisioning">Membuat Sub-Agent</ConceptLink>.
