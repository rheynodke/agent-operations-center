# Setup Embed Chat

**Embed Chat** = channel ke-4 yang lagi **on-progress development**. Beda dari Telegram/WhatsApp/Discord (yang chat lewat app eksternal), Embed Chat **disisipin langsung ke website kamu** sebagai widget — kayak Intercom atau Drift, tapi dengan agent AOC kamu.

<Callout type="warning" title="Status: Phase 1 (Private mode only)">
Saat ini cuma **Private mode** yang aktif — buat staff portal / internal tool yang udah punya login system. **Public mode** (visitor anonim, customer-facing) di-mark "coming in Phase 2" di wizard.

Jadi kalau use case kamu adalah pasang chat untuk customer di website public — **belum bisa di-deploy** sekarang. Tunggu Phase 2.
</Callout>

## Bedanya dengan channel lain

| Channel | Cara akses | Cocok untuk |
|---|---|---|
| Telegram / WhatsApp / Discord | User chat lewat app chat mereka | Personal use, mobile-first, bot publik |
| **Embed Chat** | Widget langsung di website kamu | **Staff tool / internal portal yang udah punya login** (Phase 1) — atau customer-facing widget (Phase 2 coming) |

Kelebihan Embed Chat:

- ✅ User gak perlu install / pakai app lain — chat langsung di context aplikasi yang sedang mereka pakai
- ✅ Bisa **brand sesuai company kamu** (nama, warna, welcome message, avatar)
- ✅ Kontrol penuh siapa yang bisa akses lewat **JWT visitor authentication**
- ✅ Kill switch instant kalau ada masalah

## Yang kamu butuhin (Private mode, Phase 1)

- **Website / aplikasi** kamu sendiri (staff portal, internal tool, dashboard)
- **Login system** existing yang bisa generate JWT (mis. Node.js / Python / PHP / Go backend)
- Akses untuk inject `<script>` tag ke HTML
- 1 agent yang siap dijadikan host chat

## Bikin embed baru — 5 step wizard

Klik **+ New Embed**.

<Steps>
  <Step title="Pilih mode">
    - **🔒 Embed for Internal Tool** (Private) — staff portal, ops dashboard. **Cuma ini yang available di Phase 1.**
    - **🌐 Embed for Public Site** (coming Phase 2) — anonymous customer-facing chat. Disabled.

    Klik Internal Tool → **Next**.
  </Step>

  <Step title="Pilih agent">
    Dropdown list semua agent kamu (kecuali agent public yang udah di-claim untuk channel publik). Pilih satu yang akan handle chat di embed ini.

    Recommended: agent yang punya persona match dengan use case (mis. agent "Helpdesk" buat staff portal, "Onboarding Buddy" buat new hire tool).
  </Step>

  <Step title="Origins — domain mana yang boleh load embed">
    - **Production origin** (required) — URL persis tempat embed terpasang. Contoh: `https://staff.acme.com`. Wajib HTTPS untuk production.
    - **Dev origins** (optional) — comma-separated patterns untuk testing. Contoh: `http://localhost:*, *.local, https://staging.acme.com`

    AOC akan **block load** dari domain yang gak listed — proteksi anti-clone.
  </Step>

  <Step title="Branding">
    - **Brand name** — nama yang muncul di header chat (mis. "Acme Helpdesk")
    - **Brand color** — color picker untuk accent widget (button, header, link)
    - **Welcome title** — pesan greeting saat user buka chat (default: "👋 Halo! Ada yang bisa saya bantu?")
  </Step>

  <Step title="Review & Create">
    Preview JSON config terakhir. Klik **Create Embed**.

    Setelah sukses → redirect ke detail page dengan token + signing secret yang baru di-generate.
  </Step>
</Steps>

### Tab Behavior

Atur perilaku percakapan:

- **Quick replies** — 1-click button suggestion saat chat baru terbuka (mis. "Track order saya", "Reset password")
- **Waiting text** — pesan saat agent lagi mikir (default: "Sedang mengetik...")
- **Offline message** — pesan saat embed di-disable (mode maintenance / emergency)
- **Consent text** — checkbox terms saat user pertama buka chat (optional)
- **Default language** — bahasa interface widget

### Tab Security

Tempat konfigurasi proteksi + kill switch:

**Kill switch** — 3 mode:

| Mode | Behavior |
|---|---|
| **Enabled** ✅ | Widget aktif, terima chat normal |
| **Maintenance** 🔧 | Widget tampil pesan "Sedang maintenance" — gak bisa kirim chat. Visitor existing pun di-cut. |
| **⛔ Emergency** | Widget instant disabled total. Pakai saat ada incident (data leak, abuse) — semua session mati seketika. |

**DLP (Data Loss Prevention)** — preset:

- `internal-tool-default` — strict, block leak credit card / SSN / token / sensitive PII
- `customer-service-default` — lebih permissive untuk customer-facing flow (Phase 2)

Plus **DLP allowlist patterns** — kalau preset ngeblok pattern yang sebenernya aman buat use case kamu, tambah di allowlist (1 pattern per line).

**Signing secret** (Private mode only):

Secret HMAC HS256 untuk sign visitor JWT dari host site kamu. Kalau bocor, klik **⚠ Regenerate signing secret** — semua session active langsung invalid, kamu **wajib update env** di host site dengan secret baru.

### Tab Snippet

HTML snippet siap-paste ke website kamu:

```html
<!-- AOC Embed: Acme Helpdesk -->
<script src="https://your-aoc-domain.com/embed/<embed-id>/loader.js"
        data-token="<embed-token>" defer></script>
```

Klik **Copy** → paste tepat sebelum `</body>` di template HTML.

Untuk Private mode, kamu juga butuh **JWT signing** dari backend host kamu. Snippet tab kasih contoh Node.js:

```js
// Backend kamu (Node.js example)
const jwt = require('jsonwebtoken');
const SIGNING_SECRET = process.env.AOC_EMBED_SIGNING_SECRET;

function signVisitorJwt(visitorId, name, email) {
  return jwt.sign(
    { visitor_id: visitorId, name, email },
    SIGNING_SECRET,
    { algorithm: 'HS256', expiresIn: '5m' }
  );
}

// Inject ke page (server-side render):
// <script>window.AOC_EMBED_JWT = "${signedJwt}";</script>
```

Widget loader akan baca `window.AOC_EMBED_JWT` untuk authenticate visitor ke AOC.

<Callout type="info" title="Phase 2: contoh Python / PHP / Go">
Saat ini cuma Node.js example. Phase 2 nambah Python (jose), PHP (firebase/php-jwt), Go (golang-jwt) examples.
</Callout>

## Pasang di website kamu

<Steps>
  <Step title="Copy snippet HTML">
    Dari tab Snippet → tombol Copy.
  </Step>
  <Step title="Paste sebelum </body>">
    Di template HTML kamu (mis. `_layout.html`, `Layout.tsx`, `<head>` Vue, dll). Idealnya di server-side render layer biar visitor JWT bisa di-inject langsung.
  </Step>
  <Step title="Setup JWT signing di backend (Private mode)">
    Tambah library JWT di backend. Generate signed token per visitor pakai signing secret.
    Inject token ke `window.AOC_EMBED_JWT` saat render page.
  </Step>
  <Step title="Test di dev origin dulu">
    Kalau kamu set dev origin `http://localhost:*`, jalanin app lokal — widget muncul di pojok kanan bawah. Klik untuk buka chat.
  </Step>
  <Step title="Deploy ke production">
    Pastikan production origin di config sama persis dengan URL deploy. Kalau beda, widget gak akan load (cors block).
  </Step>
</Steps>

## Quotas + rate limit

Tiap embed punya batas usage harian (default values, bisa di-edit per embed kalau perlu):

| Limit | Kenapa ada |
|---|---|
| Daily token quota | Kontrol biaya token AI per hari |
| Daily message quota | Limit total interaksi per hari |
| Rate limit per IP | Anti spam dari satu visitor / bot |
| Alert threshold % | Notifikasi saat usage approach quota (mis. 80%) |

Saat quota tercapai, widget tampil offline message + alert masuk ke dashboard.

## Retention

Default 30 hari — chat history visitor di-purge otomatis. Bisa di-edit per embed (compliance / regulatory reason).

## Disable All (panic button)

Di list page `/embeds`, tombol **⚠ Disable All** — emergency kill semua embed kamu sekaligus dengan mode `emergency`. Gunakan saat:

- Detect data leak / unauthorized access
- Bug major yang harus stop sebelum lebih banyak user kena
- Coordinated incident response

Setelah masalah resolved, re-enable per embed dari Security tab masing-masing.
