# Overview Channel

Channel binding bikin agent kamu reachable lewat **chat app eksternal**, bukan cuma dashboard. Pesan masuk dari channel di-route ke agent target, agent reply lewat channel yang sama.

<Callout type="info" title="4 channel didukung">
AOC saat ini support **Telegram**, **WhatsApp**, **Discord**, dan **Embed Chat** (widget di website kamu — Phase 1 internal tool only, Phase 2 public coming). Channel lain (Slack, Teams, Line, dll) belum tersedia.
</Callout>

## Konsep dasar

Setiap channel binding terdiri dari:

- **Channel type** — telegram / whatsapp / discord
- **Account ID** — identifier unik untuk credential set ini di per-user `openclaw.json` (default: `default`, biasanya = agent slug atau phone number)
- **Credential** — bot token (Telegram/Discord) atau session (WhatsApp)
- **Policy** — DM policy (pairing / allow / block), streaming mode, dan opsional group policy untuk Discord

## DM Policy

Mengontrol siapa yang boleh DM agent:

| Policy | Behavior |
|--------|----------|
| **pairing** | User harus pairing dulu (DM bot, dapat kode, approve di dashboard atau auto-approve) sebelum bisa kirim pesan |
| **allow** | Open — siapa pun yang DM bot bisa langsung interaksi |
| **block** | Tertutup — DM ditolak. Cuma group / specific allow-list yang bisa kirim |

## Streaming mode

Mengontrol bagaimana response agent ditampilkan di channel:

| Mode | Behavior |
|------|----------|
| **off** | Agent kirim final reply saja, satu pesan utuh |
| **partial** | Agent edit pesan progressively saat generate (visible "typing") |
| **full** | Stream raw chunk-by-chunk (paling responsif tapi noisy di channel) |

## Pairing flow (DM)

Untuk policy `pairing`:

<Steps>
  <Step title="User DM bot agent">
    Kirim pesan apapun ke bot/account agent.
  </Step>
  <Step title="Bot reply dengan kode">
    Kode 6-digit, valid 1 jam. User catat kode ini.
  </Step>
  <Step title="Approve di dashboard">
    Buka **Agent Detail → Channels**. Bagian "Pairing Requests" tampilin pending request. Klik approve.

    Alternatif: kalau kamu lagi onboarding wizard di step **Hubungkan**, wizard otomatis auto-approve first matching request — gak perlu klik manual.
  </Step>
  <Step title="User dapat akses">
    Setelah di-approve, user otomatis masuk allow-list. Sekarang mereka bisa interaksi penuh dengan agent.
  </Step>
</Steps>

## Status indicator

Tab **Channels** punya status dot kecil di header tab:

- 🟢 hijau — gateway running (channel binding aktif)
- ⚪ abu-abu — gateway offline (channel inactive sampai gateway restart)
