# System Overview

AOC (Agent Operations Center) is a full-stack web dashboard for monitoring, managing, and controlling OpenClaw agents. The platform is **multi-tenant by default** — each user gets an isolated filesystem, a dedicated gateway process, and one Master Agent that orchestrates their specialist team.

Three main processes running together:

1. **Vite SPA** in the browser — React UI reading state from Zustand stores, talking to API + WebSocket.
2. **AOC API** (Express, port `18800`) — JWT auth, RBAC, persists to SQLite in `data/aoc.db`, broadcasts WebSocket events.
3. **Per-user gateway** — separate OpenClaw process per user, lazy-spawned to port range `19000-19999`.

## Multi-Tenant Foundation

Every user gets:

- **Isolated filesystem** at `~/.openclaw/users/<id>/.openclaw/` (admin uses root `~/.openclaw/`).
- **Dedicated gateway process** with its own token, lifecycle managed by `gateway-orchestrator.cjs`.
- **Master Agent** (one per user) created during onboarding.
- **Shared skills + scripts** via symlink to admin's home — install once, propagate to all users.

Full details: <ConceptLink to="/docs/architecture/multi-tenant">Multi-Tenant Foundation</ConceptLink>.

## Tech stack

| Layer | Technology |
|-------|------------|
| Frontend | React 19, TypeScript, Vite, Tailwind v4, Zustand, React Router v7 |
| Backend | Node.js ≥20, Express 5 (CommonJS), JWT, SQLite via `sql.js` |
| Real-time | WebSocket (`ws` library), chokidar file watchers |
| Tooling | shadcn/ui, Radix primitives, Monaco Editor, Three.js (R3F) |

## What AOC is not

AOC is **not**:

- **Not an AI runtime** — agent execution happens in the OpenClaw gateway, AOC just observes and controls.
- **Not the source of truth for agent state** — all agent state lives in `~/.openclaw/`. SQLite only holds dashboard-only data (user accounts, agent profiles, projects, tasks, integrations).
- **Not a public-facing gateway** — admin's external gateway on port `18789` is systemd-managed separately; per-user gateways serve only internal AOC requests.
