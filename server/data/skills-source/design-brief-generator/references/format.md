# Output Format — Design Brief

## Filename

`outputs/YYYY-MM-DD-design-brief-{feature-slug}.md`

## Page constraint

Brief WAJIB max 2 pages. Lebih panjang = belum di-distill, return ke synthesizer.

## Required sections (in order)

1. **H1** — `# Design Brief: {Feature}`
2. **Header** — Date, Status, Inputs (PRD, research, competitor links)
3. **§1 Problem Statement** — 1 paragraph designer-readable
4. **§2 Target User & Context** — table (who, when, where, mental state)
5. **§3 Success Criteria** — measurable, numbered list
6. **§4 Constraints** — table (type, constraint, source)
7. **§5 References** — 3-5 visual examples with adopt + avoid rationale
8. **§6 Deliverables** — concrete enumerated checklist
9. **§7 Out-of-Scope** — explicit non-goals
10. **§8 Timeline & Review Points** — milestone table
11. **Sign-off** — UX Lead + PM + Designer ack

## Success criteria rules

- WAJIB measurable
- WAJIB derived dari PRD success metric atau UX research finding
- ❌ "Looks modern" / "Feels premium" / "Good UX" — not measurable

## References rules

- Minimum 3, max 5
- Per ref WAJIB: source URL/path + adopt rationale + caveats (avoid)
- ❌ "Just look at this" tanpa rationale

## Deliverables rules

- Enumerated, concrete
- Per item ada checkbox
- ❌ "Design the new flow" — open-ended invites scope creep

## Anti-pattern

- ❌ Brief > 2 pages
- ❌ Subjective success criteria
- ❌ References tanpa rationale
- ❌ Deliverables open-ended
- ❌ Skip Out-of-Scope
- ❌ Constraint missing (designer akan violate brand)
- ❌ Brief sebelum PRD lock
- ❌ No source citation untuk claim
